# Security Policy

## Reporting a vulnerability

If you discover a security vulnerability, please do not open a public issue.

Report it via one of these channels:
- Open a GitHub Security Advisory (if this repository has the feature enabled)
- Otherwise, open an issue with the title `[Security]` and avoid posting sensitive details in the public description

Please include:
- Affected file(s)
- Reproduction steps
- Impact assessment
- Suggested remediation (if available)

## Supported versions

This project is a hackathon portfolio project. We provide security fixes on a
best-effort basis for the latest commit on the main branch.

## Safe reporting practices

- Do not include API keys, secrets, or personally identifiable information.
- If the issue is urgent, mark the issue with high priority and include `security` in the title.
